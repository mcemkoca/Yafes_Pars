Yafes Pars — Copilot MCP Rehberi

Model Context Protocol · Operatör Yapay Zeka Asistanı · Sürüm 1.0

Mimari

Operatör doğal dil ile çalışır. AI (Copilot / Claude) bu isteği MCP tool call'a çevirir. MCP Server SQL Server'a doğrudan bağlanır ve sonucu AI'ya iletir. AI yanıtı kullanıcıya özetler.

Yanıt yönü: SQL Server → MCP Server → AI → Operatör

MCP Server, mevcut YafesPars.Infrastructure katmanını kullanır. Ayrı bir API çağrısı yapmaz — doğrudan veritabanına bağlanır. JWT gerektirmez, tenant_id config'den alınır.

1. Gereksinimler

✓
Sistem gereksinimleri
  • .NET 8 SDK
  • SQL Server 2019+ (YafesPars veritabanı migration 000–021 çalışmış olmalı)
  • Claude Desktop veya VS Code + GitHub Copilot

2. Projeyi Derle

1
Repo'yu klonla ve derle
git clone https://github.com/mcemkoca/Yafes_Pars.git
cd Yafes_Pars

dotnet build backend/src/YafesPars.McpServer/YafesPars.McpServer.csproj
2
Test et — stdio'da başlat
dotnet run --project backend/src/YafesPars.McpServer/YafesPars.McpServer.csproj

Hata yoksa cursor bekler (MCP sunucu hazır, stdin bekliyor). Ctrl+C ile çık.

3. Konfigürasyon

!
appsettings.json düzenle

Dosya: backend/src/YafesPars.McpServer/appsettings.json

{
  "ConnectionStrings": {
    "YafesPars": "Server=SUNUCU_ADI;Database=YafesPars;
                  User Id=kullanici;Password=sifre;
                  TrustServerCertificate=True;"
  },
  "McpServer": {
    "TenantId": "GERÇEK-TENANT-GUID-BURAYA"
  }
}
TenantId'yi veritabanındaki gerçek tenant GUID'i ile değiştir.
Sorgu: SELECT tenant_id, name FROM core.Tenant

4a. Claude Desktop ile Kullanım

1
Claude Desktop config dosyasını aç
# Windows
%APPDATA%\Claude\claude_desktop_config.json

# macOS
~/Library/Application Support/Claude/claude_desktop_config.json
2
mcpServers bloğunu ekle
{
  "mcpServers": {
    "yafes-pars": {
      "command": "dotnet",
      "args": [
        "run",
        "--project",
        "C:\\Yafes_Pars\\backend\\src\\YafesPars.McpServer\\YafesPars.McpServer.csproj",
        "--no-launch-profile"
      ],
      "env": {
        "ConnectionStrings__YafesPars": "Server=localhost;Database=YafesPars;Integrated Security=True;TrustServerCertificate=True;",
        "McpServer__TenantId": "TENANT-GUID-BURAYA"
      }
    }
  }
}
3
Claude Desktop'ı yeniden başlat

Sol altta 🔧 simgesi görünüyorsa MCP bağlantısı başarılı.

4b. VS Code + GitHub Copilot ile Kullanım

1
Proje kökünde mcp.json zaten hazır

Repo'da mcp.json dosyası mevcut. VS Code GitHub Copilot bunu otomatik algılar.

2
VS Code settings.json'a ekle (gerekirse)
{
  "github.copilot.chat.mcp.enabled": true
}
3
Copilot Chat'i aç → Agent modu seç

Ctrl+Shift+I → üstten Agent modunu seç → yafes-pars tool'ları görünür.

4c. GitHub Copilot Extension (İleri Düzey)

GitHub Copilot Extension, MCP yerine Copilot API'sini kullanır. Sunucu bir webhook endpoint'e ihtiyaç duyar (Azure App Service veya ngrok).
1
Mevcut API'yi Copilot Extension endpoint'i olarak kullan
# Yafes Pars API'yi başlat
dotnet run --project backend/src/YafesPars.Api/YafesPars.Api.csproj

# ngrok ile dışa aç (geliştirme ortamı)
ngrok http 5000
2
GitHub → Settings → Developer Settings → GitHub Apps → New

Callback URL: ngrok URL'in + /api/copilot/callback
Copilot: ✅ Copilot Chat Enabled

Bu seçenek MCP'ye göre daha karmaşık. Üretim ortamı için MCP (4a veya 4b) önerilir.

Azure'a Deploy

1
MCP Server'ı Container Image olarak paketle
cd backend/src/YafesPars.McpServer
dotnet publish -c Release -o publish/

# Docker image oluştur
docker build -t yafespars-mcp:latest -f Dockerfile.mcp .
2
Azure Container Instance olarak deploy et
az container create \
  --resource-group rg-yafespars-prod \
  --name yafespars-mcp \
  --image ghcr.io/mcemkoca/yafes_pars-mcp:latest \
  --environment-variables \
    McpServer__TenantId="TENANT-GUID" \
  --secure-environment-variables \
    ConnectionStrings__YafesPars="..." \
  --restart-policy Always
3
Claude Desktop'ı remote MCP ile yapılandır
{
  "mcpServers": {
    "yafes-pars": {
      "url": "https://yafespars-mcp.westeurope.azurecontainer.io/mcp",
      "transport": "sse"
    }
  }
}
SSE (Server-Sent Events) transport ile MCP sunucu remote çalışabilir. Azure Container Instance'a public IP verilmeli.

Sorun Giderme

MCP sunucu başlamıyor

TenantId geçerli GUID formatında değil. appsettings.json'ı kontrol et.

SELECT tenant_id, name FROM core.Tenant -- doğru GUID'i bul

Bağlantı hatası

Connection string kontrol et:

sqlcmd -S SUNUCU -d YafesPars -Q "SELECT 1"

Claude Desktop tool'ları görmüyor

Log dosyasını incele:

# Windows
%APPDATA%\Claude\logs\mcp-server-yafes-pars.log

ModelContextProtocol NuGet bulunamadı

dotnet nuget add source https://api.nuget.org/v3/index.json
